PyTorch for Intel XPU 在虚拟环境中导入时抛出Error 127

发表于 2026-05-30

调试记录 — PyTorch for Intel XPU 在虚拟环境中导入时抛出Error 127

  • 日期：2026-05-30

1. 环境

  • 操作系统：Windows 11
  • Python / 运行时：Python 3.13.0
  • 虚拟环境 / venv：python -m venv llm-env
  • 包版本：（例如，torch==x.y.z）各种版本
  • 硬件（GPU/CPU）：Intel Arc A770

2. 问题描述

系统 Python 环境中已安装 torch 2.6.0+xpu，工作正常。但在未继承系统 Python site-packages 的虚拟环境中无法使用。
始终抛出以下错误：

3. 调查过程

首先，这个虚拟环境中的 Dependencies 工具显示 sycl8.dll 是 c10_xpu.dll 的缺失依赖项。
然后，Everything 发现 sycl8.dll 位于 C:\Users\whye\llm-env\Library\bin\sycl8.dll。设置 PATH=C:\Users\whye\llm-env\Library\bin;%PATH% 后，Dependencies 不再显示缺失项；然而，虚拟环境中的 import torch 仍然报错。

如报错消息所示，报错发生在 C:\Users\whye\llm-env\Lib\site-packages\torch\__init__.py，因此可以使用 Python Debugger 调试代码。为了调试第三方包，需要在 VS Code 的 launch.json 中添加 "justMyCode": false：

随后，我们可以看到 res = kernel32.LoadLibraryExW(dll, None, 0x00001100) 在加载 C:\Users\whye\llm-env\Lib\site-packages\torch\lib\c10_xpu.dll 时返回 None， ctypes.WinError(last_error) 返回 127。

标志 0x00001100 代表以下标志的组合：

到这一步，我没有进一步的线索可以继续了。幸运的是，WinDbg 可以到内核的错误消息，并提供额外的信息。
首先，我以管理员权限打开了命令提示符，然后运行了 gflags /i python.exe +sls 来启用 DLL 加载的日志记录。
接下来，我使用 C:\Users\whye\llm-env\Scripts\activate 激活了 venv。
然后，我运行了 "C:\Program Files (x86)\Windows Kits\10\Debuggers\x64\windbg.exe" -o python 来在 WinDbg 中启动 Python。输入 g 让 Python 执行，并在 Python 窗口中运行了 import torch 来触发错误。WinDbg 中出现的以下日志：

最后，这些日志揭示了根本原因：LdrpMinimalMapModule - ENTER: DLL name: C:\Users\whye\AppData\Local\Programs\Python\Python313\Library\bin\sycl8.dll。来自系统 Python 环境的 sycl8.dll 被加载了，导致 sycl8.dll 和 c10_xpu.dll 版本之间不匹配。
在系统 Python 环境中，pip list 显示安装了多个 SYCL 运行时扩展：

最终，在从系统 Python 环境中卸载这些包后，尤其是卸载 intel-sycl-rt，torch 可以在虚拟环境中成功导入。

4. 根本原因（如已确定）

DLL 与系统环境冲突

5. 修复 / 解决方案

从系统 Python 环境中卸载对应的包
