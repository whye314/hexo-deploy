Debugging Record — PyTorch for Intel XPU Throws Error 127 When Imported in Venv

  • Date: 2026-05-30

1. Environment

  • OS: Windows 11
  • Python / runtime: Python 3.13.0
  • Virtualenv / venv: python -m venv llm-env
  • Package versions: (e.g., torch==x.y.z) various
  • Hardware (GPU/CPU): Intel Arc A770

2. Problem Description

The system Python environment has torch 2.6.0+xpu installed and works fine. However, it doesn’t work in a venv without inheriting site-packages from the system Python.
This error is always thrown:

3. Investigation Performed

First, Dependencies in this venv indicated that sycl8.dll was missing as a dependency of c10_xpu.dll.
Then, Everything found that sycl8.dll was located at C:\Users\whye\llm-env\Library\bin\sycl8.dll. After setting PATH=C:\Users\whye\llm-env\Library\bin;%PATH%, Dependencies no longer showed any red flags; however, import torch still failed in the venv.

As the error message indicated, the error occurred in C:\Users\whye\llm-env\Lib\site-packages\torch\__init__.py, so we could debug the code using the Python Debugger. For debugging third-party packages, adding "justMyCode": false to launch.json in VS Code was required:

Subsequently, we’d see that res = kernel32.LoadLibraryExW(dll, None, 0x00001100) returns None and ctypes.WinError(last_error) returns 127 for loading C:\Users\whye\llm-env\Lib\site-packages\torch\lib\c10_xpu.dll.

At this point, I had no further clues to investigate. Fortunately, WinDbg allowed me to see the error messages from the kernel, which provided additional insights.
First, I opened Command Prompt with administrator privileges, then ran gflags /i python.exe +sls to enable logging for DLL loading.
Next, activate the venv with C:\Users\whye\llm-env\Scripts\activate.
Next, I ran "C:\Program Files (x86)\Windows Kits\10\Debuggers\x64\windbg.exe" -o python to start Python in WinDbg. I then pressed g to let Python execute, and ran import torch in the Python window to trigger the error. The following logs appeared in WinDbg:

Finally, these logs revealed the root cause: LdrpMinimalMapModule - ENTER: DLL name: C:\Users\whye\AppData\Local\Programs\Python\Python313\Library\bin\sycl8.dll. The sycl8.dll from the system Python environment was being loaded instead, creating a mismatch between the sycl8.dll and c10_xpu.dll versions.
In the system Python environment, pip list showed that several SYCL runtime extensions were installed:

Finally, torch could be successfully imported in the venv after uninstalling these packages, especially the intel-sycl-rt.

4. Root Cause (if determined)

DLL conflicts with system environment

5. Fix / Workaround

Uninstalling corresponding packages from the system Python environment
